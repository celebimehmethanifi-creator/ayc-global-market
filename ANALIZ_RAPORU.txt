[WEB-KG-01] JWT SECRET HARDCODED FALLBACK — KRİTİK
Dosya: apps/web/app/api/v1/_lib/auth.ts (satır 4-6)
----------------------------------------------------------------------
  const SECRET = new TextEncoder().encode(
    process.env.JWT_SECRET || "ayc-market-secret-2026-xk9m2p"
  );

JWT_SECRET env değişkeni tanımlı değilse "ayc-market-secret-2026-xk9m2p"
kullanılıyor. Bu değer kaynak kodda görünüyor yani herkes bu değeri
biliyor. Herhangi biri bu anahtarla keyfi JWT imzalayabilir.
JWT_SECRET ZORUNLU hale getirilmeli, yoksa uygulama başlamamalı.

[WEB-KG-02] TEST HESAPLARI PLAIN-TEXT ŞİFRELERLE HARDCODED — KRİTİK

  const TEST_ACCOUNTS = {
    "elite@aycmarket.com": { password: "AycElite2026!", tier: "elite", balance: 100000 },
    "pro@aycmarket.com":   { password: "AycPro2026!",   tier: "pro",   balance: 25000  },
    "free@aycmarket.com":  { password: "AycFree2026!",  tier: "free",  balance: 10000  },
    "admin@aycmarket.com": { password: "AycAdmin2026!", tier: "elite", balance: 500000 },
    ...

Bu hesaplar:
  - Üretim kodunda çalışıyor (ortam değişkeniyle devre dışı bırakılmıyor)

[WEB-KG-03] TEST HESABI TOKEN'I İMZASIZ BASE64 — KRİTİK
Dosya: apps/web/app/api/v1/auth/login/route.ts (satır 37-38)
----------------------------------------------------------------------
  const token = Buffer.from(JSON.stringify(tokenPayload)).toString("base64url");

Test hesapları için gerçek JWT imzalanmıyor, sadece JSON base64
encode ediliyor. Bu token'ı herkes kendisi oluşturabilir:
  1. İstediği payload'ı JSON'a çevir
  2. base64url ile encode et
  3. API'a gönder

Gerçek hesaplar için signAccess() kullanılıyor ama test hesapları
imzasız token alıyor. Frontend bu farkı ayırt edemiyor.

[WEB-KG-04] WEBHOOK SECRET YOKSA HEPSİ KABUL EDİLİYOR — KRİTİK

  function verifySignature(body, signature): boolean {
    if (!WEBHOOK_SECRET) return true; // skip verification if no secret set
    ...
  }

LEMON_WEBHOOK_SECRET env değişkeni tanımlı değilse imza doğrulaması

[WEB-KG-05] KULLANICI VERİLERİ IN-MEMORY GLOBALTHIS'TE — KRİTİK
Dosyalar:
  apps/web/app/api/v1/_lib/auth.ts  → __AYC_USERS_CACHE
  apps/web/app/api/v1/alarms/route.ts → __AYC_ALARMS

Kullanıcı hesapları, alarmlar ve portföy pozisyonları Next.js

  - Restart = tüm kullanıcı verisi kayıp
Edge Config'e yedekleme var ama sadece kullanıcı kayıtları için,
alarmlar ve pozisyonlar için yok.

[WEB-KG-06] CREDENTIAL TOKEN İÇİNDE HASHED ŞİFRE — YÜKSEK
Dosya: apps/web/app/api/v1/_lib/auth.ts (signCredential fonksiyonu)
----------------------------------------------------------------------
  return new SignJWT({ ..., hp: user.hashedPassword, type: "credential" })
    .setExpirationTime("365d")
    .sign(SECRET)

Hashed şifre 365 gün geçerli bir JWT içine konulmuş. Bu token:
  - Client-side localStorage'da saklanıyor
  - 365 gün boyunca geçerli
  - XSS saldırısıyla çalınırsa kullanıcı hesabına erişim sağlanabilir

[WEB-KG-07] ACCESS TOKEN 7 GÜN GEÇERLİ — YÜKSEK
Dosya: apps/web/app/api/v1/_lib/auth.ts (signAccess fonksiyonu)
----------------------------------------------------------------------
  .setExpirationTime("7d")

Standart güvenlik pratiği: access token 15 dakika - 1 saat.
7 günlük access token çalınırsa 7 gün boyunca geçerli.
